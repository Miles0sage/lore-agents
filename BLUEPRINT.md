### 1B. Hash-Chained Audit Log (2 days) — THE COMPLIANCE PRIMITIVE

Every `PolicyResult` from `AgentHypervisor` → append-only log, hash-chained.
Each entry includes SHA-256 of previous entry. Tamper-evident without blockchain.

```python
# phalanx/audit.py
class AuditLog:
    def append(self, event: PolicyResult) -> str:
        """Returns the hash of this entry (chain link)."""

    def verify(self) -> bool:
        """Verify the entire chain is intact."""

    def export_pdf(self, period: str, frameworks: list[str]) -> bytes:
        """Generate compliance PDF mapped to OWASP/SOC2/EU-AI-Act."""
```

PDF contents:
- Fleet operation summary (N actions, M failures, K rules active)
- Compliance mapping to OWASP Agentic Top 10
- Anomalies detected and how resolved
- Rules updated (with PR numbers and approval timestamps)
- Cryptographic chain verification proof

## Phase 2 — Moat Building (Weeks 4-8)

### 2A. SELF-REFINE Verbal Retry Loop (3 days) — THE MOAT

Right now `@watch` logs failures passively. The steal:
When `@watch` catches a failure → critique prompt → retry with verbal feedback → store reflection.

```python
@watch(max_reflexion_trials=3, critique_model="gpt-4o-mini")
def my_agent_fn(input: str) -> str: ...
# On failure: critique → refine → retry → store (failure, critique, outcome)
# compile_rules() clusters on verbal critiques, not just tracebacks
```

`ReflexionMemory` store: `(failure_json, verbal_critique, retry_success)`.
`compile_rules()` gains semantic signal. Rules become meaningful, not just keywords.

### 2B. PolicyProvider ABC (2 days) — ARCHITECTURE UNLOCK

Makes phalanx pluggable into existing enterprise policy engines.

```python
class PolicyProvider(ABC):
    def evaluate(self, fn_name: str, args: dict, trust_score: int) -> PolicyDecision: ...

# Ships with:
class RulesFilePolicyProvider(PolicyProvider): ...   # reads SAFETY_RULES.md
class OPAPolicyProvider(PolicyProvider): ...         # calls OPA sidecar
class AllowAllProvider(PolicyProvider): ...          # for testing
```

### 2C. AgentOps Span Hierarchy (3 days) — DEBUGGABILITY

Every `@watch` failure gains: `session_id → agent_id → operation_id → tool_called`.
`compile_rules()` generates context-aware rules:
"NEVER call tool X from agent Y when in task Z."

### 2D. Rebuff Prompt Injection Detection (2 days) — SECURITY BASELINE

50-line heuristic scorer, zero LLM calls, sub-millisecond.
Combinatorial keyword variants + canary token injection.
If model leaks canary token → `CANARY_LEAK: CRITICAL` failure recorded.

### 2E. NeMo 5-Rail Taxonomy (2 days) — RULE STRUCTURE

Tag every failure: `input | dialog | execution | output | retrieval`.
SAFETY_RULES.md becomes sectioned by rail type. Self-documenting rules.
Users immediately know "this is an execution rail violation" vs "input rail."

### 2F. AutoGen Approval Gate (4 days) — ENTERPRISE BLOCKER SOLVER

Ring-0 tools PAUSE execution, emit `ApprovalRequest`, await explicit approval.
```python
@watch(ring=0, approval_mode="always_require")
def delete_database(name: str): ...
# CLI: phalanx approve req_abc123
# or via Slack bot / HTTP webhook
```
Irreversible actions (delete, deploy, send email) get human checkpoint.
